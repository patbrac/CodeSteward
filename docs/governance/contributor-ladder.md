# Contributor ladder

Roles recognize sustained stewardship and assign clear duties; they are not status rewards.

| Role | Typical capabilities | Required behavior |
|---|---|---|
| Contributor | Issues, pull requests, discussion, tests, docs | DCO sign-off, Code of Conduct, authorized/synthetic data, responsive review collaboration |
| Area reviewer | Review/request approval in named areas | Demonstrated domain judgment, timely evidence-based reviews, escalation of security/compatibility concerns |
| Maintainer | Merge, roadmap/governance decisions, role nominations | Project-wide stewardship, conflict disclosure, contributor support, operational ownership |
| Security responder | Private advisory access, embargo coordination | Least privilege, confidentiality, response targets, synthetic reproduction, coordinated disclosure |
| Release manager | Release workflow approval and artifact promotion | Native matrix, checksum/SBOM/attestation, rollback, notes, support/deprecation verification |

Promotion evidence is quality and consistency over time, not a numeric commit threshold. Review, documentation, incident prevention, mentoring, and cross-platform fixture work count equally with code. Employer, commercial-customer status, or use of automation does not grant authority.
