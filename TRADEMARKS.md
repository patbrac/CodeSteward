# Trademark policy

The Apache-2.0 license covers copyright and patent permissions for Code Steward software; it does not grant trademark rights. “Code Steward” and project logos identify the official project and its releases. This policy is not a statement that a mark is registered and is not legal advice.

## Uses that do not require permission

You may:

- truthfully state that software uses, integrates with, or was derived from Code Steward;
- link to the official repository and reproduce unmodified screenshots for commentary, education, or news;
- say “compatible with Code Steward” when the compatibility claim is accurate and not more prominent than your own product name;
- use the name in package metadata solely to describe a dependency or protocol;
- refer to the project in talks, articles, books, issue reports, and comparative evaluations.

These uses must not imply sponsorship, certification, partnership, or endorsement.

## Forks and modified distributions

You may exercise the Apache-2.0 rights to fork and distribute the code. A materially modified distribution must use a distinct primary name and visual identity, clearly state that it is unofficial, and identify its changes. It may use wording such as “derived from Code Steward” or “compatible with Code Steward.”

Do not name a fork, hosted scanner, company, package, domain, social account, conference, or service “Code Steward” or use a confusingly similar name or logo. Do not remove attribution required by `LICENSE` and `NOTICE`.

## Uses requiring written permission

Written permission is required to:

- use a project logo as your product, service, company, or event identity;
- use the name in a domain, application-store listing, paid-search keyword, or commercial service branding in a way likely to imply an official offering;
- describe a product as “official,” “certified,” or “approved” by Code Steward;
- produce merchandise primarily displaying the project marks, except for incidental, non-commercial community use.

Requests may be opened as a non-sensitive repository issue describing the proposed use. Do not assume silence is consent.

## Fair use and law

Nothing here restricts nominative fair use, criticism, commentary, or other uses permitted by applicable law. The policy should be applied reasonably to prevent confusion, not to suppress accurate discussion or interoperable software.

Maintainers may ask a user to correct confusing branding before escalating. Questions about whether a use is permitted should be resolved before launch; obtain independent legal advice when the stakes warrant it.
